# 6.6 Alternativas de gestion de secretos e identidad

La evaluacion de secretos e identidad partio de un estado con alta carga manual, riesgo de exposicion de credenciales y modelo de autenticacion heredado. El objetivo fue identificar una alternativa escalable, auditable y compatible con automatizacion/GitOps.

Esta seccion toma al proyecto APIM como referencia modeladora de la reingenieria OpenShift; por eso las conclusiones se consideran base de diseno y no cierre automatico para todos los dominios de plataforma.

## Alternativas evaluadas

- **HashiCorp Vault (con operadores de integracion en OpenShift)**:
  - Fortalezas: centralizacion de secretos, politicas de acceso, auditoria y rotacion.
  - Modalidades analizadas: Vault Secrets Operator (VSO) y patrones de inyeccion via webhook/sidecar.
  - Limitaciones: complejidad de implementacion, definicion de ownership y costo operativo/licenciamiento.
  - Estado: opcion prioritaria en proceso de implementacion VSO.

- **External Secrets Operator (integrado con Vault)**:
  - Fortalezas: sincronizacion declarativa de secretos hacia namespaces y menor friccion para equipos de desarrollo.
  - Limitaciones: agrega dependencia operacional adicional y requiere backend robusto de secretos.
  - Estado: alternativa complementaria bien alineada con modelo GitOps.

- **CyberArk (para este caso de uso especifico)**:
  - Hallazgo: cobertura parcial para necesidades de configuracion/secreto declarativo en Kubernetes.
  - Estado: descartado para el patron objetivo de plataforma.

## Alternativas de identidad y autenticacion asociadas

- **Modelo actual basado en API keys estaticas**:
  - Riesgo: revocacion limitada, trazabilidad debil y baja alineacion regulatoria.

- **Modelo objetivo basado en OAuth2/JWT + mTLS + identidad de workload**:
  - Beneficio: mejor control de acceso, expiracion/revocacion, menor riesgo de credenciales compartidas.
  - Implicancia: migracion gradual por dominios/equipos para reducir riesgo de corte.

## Criterios de comparacion

- Seguridad efectiva (rotacion, revocacion, least privilege, trazabilidad).
- Integracion con AD/RBAC y stack de seguridad corporativo.
- Operabilidad en OpenShift con automatizacion.
- Costo de adopcion y complejidad de transicion desde el modelo actual.
