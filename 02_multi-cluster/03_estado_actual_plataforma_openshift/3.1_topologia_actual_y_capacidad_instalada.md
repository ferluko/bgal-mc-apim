# 3.1 Topologia actual y capacidad instalada

La plataforma actual de OpenShift se organiza principalmente sobre un esquema monolitico, con un cluster productivo de gran escala que concentra la operacion critica del banco. En las reuniones tecnicas se reporta una dimension superior a 100 nodos, con miles de pods y cientos de namespaces, y una topologia stretch entre dos sitios principales (Plaza/PGA y Matriz/CMZ). Esta configuracion centralizada convive con una instancia de contingencia en esquema activo-standby para el dominio APIM y con entornos no productivos para desarrollo/pruebas.

La distribucion de cargas muestra una fuerte concentracion en servicios transaccionales y de integracion: microservicios internos, APIs externas, integraciones con core y legacy, y un volumen relevante de procesos batch. El trafico reportado se ubica en torno a 8 mil millones de requests mensuales, con predominio del trafico interno (east-west) frente al externo (north-south), lo que tensiona capacidades de red, gateways y operacion diaria.

Desde capacidad instalada, la evidencia consolidada indica un parque operativo de referencia con mas de 100 nodos, mas de 10.000 pods y mas de 600 namespaces, junto con restricciones de sobrecompromiso de recursos en nodos productivos. Para componentes de exposicion, se mencionan 7 ingress controllers con aproximadamente 400 rutas cada uno y una base de alrededor de 2.200 APIs productivas, lo que confirma alta densidad operacional sobre el entorno actual.
