# 3.2 Modelo operativo (dia 0, dia 1, dia 2)

## Dia 0

- El aprovisionamiento inicial de cluster requiere coordinacion previa de infraestructura base (red, VIPs, DNS, almacenamiento y accesos).
- Se reporta necesidad operativa de 2-3 VIPs por cluster para API y trafico de ingreso, con dependencias en equipos de red/comunicaciones.
- El despliegue se realiza sobre instalacion IPI con customizaciones operativas del banco.
- Persisten pasos manuales y tickets para habilitar componentes de red y balanceo, lo que condiciona tiempos de provision.
- La automatizacion existe de forma parcial en scripts/playbooks, pero no elimina la dependencia de aprobaciones y tareas manuales externas.

## Dia 1

- La configuracion base incluye integraciones de plataforma con F5, DNS corporativo y componentes de seguridad/identidad.
- Se utilizan playbooks para acelerar alta de objetos de red (virtual servers, pools, monitores), aunque el circuito sigue sujeto a coordinacion interequipos.
- Se establecen configuraciones iniciales de seguridad, RBAC y politicas base por entorno.
- La preparacion de componentes de exposicion de servicios (ingress, certificados, rutas) combina automatizacion tecnica con validaciones manuales.
- La estandarizacion de configuraciones entre sitios existe, pero con evidencias de sincronizaciones operativas no totalmente declarativas.

## Dia 2

- La operacion continua se estructura en tracks diferenciados: infraestructura/nodos, seguridad/RBAC y DevOps/release management.
- El modelo actual incorpora practicas GitOps en algunos dominios, pero convive con procesos manuales para sincronizacion y gestion de cambios.
- La continuidad operativa en APIM mantiene pasos manuales relevantes (comparacion de configuraciones, publicacion y suscripcion de APIs entre sitios).
- El failover en componentes criticos reporta dependencia de acciones manuales (por ejemplo ajustes de DNS), sin automatizacion end-to-end en todos los casos.
- Se observan dependencias operativas persistentes en accesos privilegiados, tickets de red y procesos de certificados, que aumentan carga de operacion diaria.
