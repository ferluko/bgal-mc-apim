# 3.3 Networking, ingress/egress y exposicion de servicios

El modelo de red actual se apoya en una topologia stretch entre dos sitios, con un cluster OpenShift monolitico que concentra la mayoria del trafico de negocio. Para exposicion y ruteo, la cadena de ingreso combina componentes perimetrales (F5/Fortinet), ingress sobre OpenShift (HAProxy) y APIM (3scale/Apicast), tanto para trafico externo como para gran parte de la comunicacion interna entre servicios.

En la practica, la exposicion de servicios internos y externos no esta completamente desacoplada por tipo de trafico. Las reuniones describen un patron de hairpinning donde llamadas service-to-service salen del cluster hacia balanceadores externos y reingresan al cluster para resolver autorizacion/ruteo, agregando latencia y complejidad operativa. Este patron impacta especialmente por el peso del trafico east-west sobre el total mensual.

Desde el punto de vista operativo, networking e ingress/egress presentan alta dependencia de procesos manuales: provisioning de VIPs, cambios en DNS corporativo, certificados y habilitaciones de red. Aunque existen automatizaciones parciales (playbooks/scripts), la ejecucion real sigue sujeta a tickets, ventanas y coordinacion con equipos externos a plataforma.

- **Modelo de red actual:** cluster monolitico con red extendida entre sitios y esquema de contingencia activo-standby en componentes clave de APIM.
- **Ingress principal:** HAProxy/OpenShift ingress controllers, con reporte de varios controladores y alta densidad de rutas por controlador.
- **Egress y salida controlada:** gobernada por politicas de red y componentes perimetrales corporativos.
- **Exposicion externa:** F5/Fortinet + DMZ + ingress OpenShift + Apicast para APIs north-south.
- **Exposicion interna:** trafico east-west mayoritario cursando por APIM y balanceadores externos en varios flujos.
- **Limitacion estructural:** hairpinning con latencia adicional reportada (25-50 ms por salto) y mayor probabilidad de bottlenecks.
- **Limitacion de observabilidad:** 3scale/Apicast interrumpe trazabilidad end-to-end en parte del flujo.
- **Limitacion operativa:** dependencia de procesos manuales para DNS, certificados y componentes de balanceo.
- **Limitacion de escalado:** concentracion de rutas y politicas en capa de ingress/APIM con impacto sobre crecimiento operativo.
