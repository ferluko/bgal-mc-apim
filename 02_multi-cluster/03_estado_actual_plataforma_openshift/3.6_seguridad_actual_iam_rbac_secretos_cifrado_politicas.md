# 3.6 Seguridad actual (IAM/RBAC, secretos, cifrado, politicas)

## Identidad y acceso (IAM/RBAC)

La autenticacion de plataforma se integra con Active Directory corporativo y combina usuarios de directorio con cuentas tecnicas para operacion. En las reuniones se reportan desalineaciones de privilegios (incluyendo cuentas con permisos elevados para tareas de DevOps) y necesidad de mayor estandarizacion RBAC por rol y entorno. Tambien se observa dependencia operativa de cuentas compartidas en algunos flujos, con impacto en segregacion de funciones y trazabilidad.

## Gestion de secretos

El estado actual combina secretos nativos de Kubernetes/OpenShift con procesos manuales en pipelines y placeholders a completar por equipos. Para cargas criticas, esta modalidad incrementa friccion operativa y riesgo de inconsistencia entre entornos/sitios. Las conversaciones tecnicas muestran transicion en curso hacia un esquema mas centralizado con Vault/External Secrets, pero en el estado actual todavia conviven practicas heterogeneas de gestion de credenciales.

## Cifrado y credenciales

Se reportan controles vigentes y pendientes: hay lineamientos para endurecer tiempos de vida de tokens y reducir exposición de credenciales estaticas, mientras que el cifrado de ETCD aparece como requisito explicitado para nuevos clusters con implementacion no uniforme en toda la base instalada. En APIM se identifican patrones de autenticacion legacy (API keys estaticas) que introducen dificultad de revocacion y observaciones de auditoria.

## Politicas de red y controles de seguridad

Existen Network Policies de aislamiento basico entre proyectos/namespaces, con reglas de entrada necesarias para trafico de APIs, pero con cobertura de egress no completamente cerrada en el modelo actual. A nivel operativo, la aplicacion de politicas y limites de recursos se encuentra activa pero con margen de mejora en granularidad y gobierno de objetos. En conjunto, la postura actual de seguridad es funcional para operacion diaria, aunque con brechas de homogeneidad, automatizacion y control fino multicluster.
