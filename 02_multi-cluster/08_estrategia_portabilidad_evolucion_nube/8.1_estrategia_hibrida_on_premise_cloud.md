# 8.1 Estrategia hibrida on-premise + cloud

La estrategia adoptada para Galicia es un modelo hibrido gobernado: mantener on-premise como base de continuidad para cargas criticas y habilitar cloud como vector de escalabilidad, resiliencia y velocidad de entrega. El enfoque evita migraciones big bang y prioriza evolucion por oleadas con control de riesgo.

## Estrategia de adopcion

- **On-premise como base operativa:** sostiene cargas de alta criticidad, fuerte acoplamiento al core y restricciones regulatorias exigentes.
- **Cloud como capacidad de expansion:** incorpora elasticidad, aislamiento de dominios y aceleracion para nuevos servicios digitales.
- **Operacion unificada:** misma disciplina de gobierno, seguridad y observabilidad para ambos entornos.
- **Transicion progresiva:** coexistencia controlada on-prem/cloud hasta estabilizar cada ola de migracion.

## Stack habilitador de la estrategia

- OpenShift multicluster como plano de ejecucion comun.
- ACM + ArgoCD para gobierno de flota y sincronizacion declarativa.
- Service mesh east-west (Cilium Mesh Enterprise) + API gateway north-south (linea Gloo) para separar traficos y escalar.
- External DNS/F5 GTM para enrutamiento y failover automatizado.
- Vault + External Secrets para gestion corporativa de credenciales.
- OpenTelemetry + Prometheus/Grafana + eBPF para trazabilidad tecnica integral.

## Cambios estructurales a ejecutar

- Segmentar progresivamente el cluster monolitico en clusters por dominio/criticidad.
- Definir contratos de plataforma portables (runtime, red, seguridad, observabilidad).
- Pasar de operacion por tickets a flujos declarativos para day 0/day 1/day 2.
- Introducir failover automatizado inter-sitio y preparar extension a cloud publico.

## Beneficios esperados

- Reduccion del blast radius y mayor continuidad ante fallas de plataforma.
- Mayor velocidad para habilitar cargas cloud-ready sin comprometer estabilidad actual.
- Mejor previsibilidad de costos y decisiones de placement por dominio.
- Menor deuda operativa al unificar stack y gobierno entre entornos.

## Criterios de avance

- Cada ola debe demostrar seguridad, observabilidad y recuperacion equivalentes entre origen y destino.
- Ninguna carga avanza si no cumple requisitos de cumplimiento y trazabilidad definidos por el programa.
- El avance a cloud se mide por capacidades operativas transferidas, no solo por workloads movidos.
