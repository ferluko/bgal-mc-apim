# 8.5 Dependencias criticas para adopcion cloud

La adopcion cloud se ejecuta como programa de habilitadores. En lugar de presentar dependencias como restricciones estaticas, se definen como backlog de capacidades que habilita cada ola de migracion con control tecnico y regulatorio.

## Habilitadores tecnicos a construir

- **Conectividad hibrida:** enlaces privados y ruteo consistente entre data centers y cloud.
- **Red y DNS automatizados:** resolucion dinamica, failover y enrutamiento declarativo.
- **Identidad y secretos:** IAM federado + Vault/ESO para credenciales multientorno.
- **Datos y continuidad:** patrones de backup, replicacion y DR para cargas stateful.
- **Observabilidad unificada:** visibilidad end-to-end para operar cargas distribuidas.

## Habilitadores organizacionales

- Modelo de ownership transversal entre plataforma, seguridad, redes, operaciones y desarrollo.
- Capacidad operativa 7x24 para incidentes cross-entorno con runbooks unificados.
- Gobierno de cambios con ventanas y criterios de riesgo por dominio.
- Plan de formacion para reducir dependencia de conocimiento concentrado.

## Habilitadores regulatorios

- Evidencia auditada de accesos, cambios y operaciones entre entornos.
- Controles de segregacion de funciones y aprobacion trazable.
- Politicas de residencia/custodia de datos por clasificacion.
- Pruebas periodicas de continuidad con evidencia verificable.

## Plan de ejecucion de dependencias

- **Paso 1:** cerrar habilitadores de red, identidad y secretos.
- **Paso 2:** certificar observabilidad y trazabilidad regulatoria.
- **Paso 3:** habilitar pilotos cloud con cargas cloud-ready.
- **Paso 4:** escalar por oleadas segun cumplimiento de criterios go/no-go.

## Beneficios esperados

- Menor riesgo de migraciones fallidas por precondiciones incompletas.
- Mayor velocidad de adopcion cloud al convertir bloqueos en backlog accionable.
- Mejor control ejecutivo del avance por capacidades, no solo por fechas.
- Continuidad y cumplimiento sostenidos durante toda la transicion.
