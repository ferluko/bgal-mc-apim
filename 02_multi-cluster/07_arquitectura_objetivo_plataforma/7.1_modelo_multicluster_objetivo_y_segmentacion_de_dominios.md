# 7.1 Modelo multicluster objetivo

El modelo objetivo de plataforma se define sobre una arquitectura multicluster para reducir riesgo sistemico y desacoplar el crecimiento de cargas. Esta definicion toma como referencia directa el proyecto APIM (caso modelador), y extiende sus decisiones de arquitectura al programa de reingenieria OpenShift.

La segmentacion propuesta separa dominios por criticidad y tipo de servicio, evitando que una falla de infraestructura o de una linea de negocio impacte al resto de la operacion. El objetivo es pasar de un cluster monolitico a una flota de clusters con responsabilidades claras, incluyendo clusters aplicativos, clusters de servicios compartidos y componentes centrales de gobierno.

## Principios del modelo

- Aislamiento por dominio para reducir blast radius.
- Estandar base por cluster para simplificar operacion y ciclo de vida.
- Topologia activo-activo entre sitios donde aplique al negocio.
- Sincronizacion declarativa de configuraciones como fuente unica de verdad.
- Separacion de control plane y data plane para resiliencia y escalado independiente.

## Segmentacion objetivo

- Clusters de negocio para cargas aplicativas de tribus o dominios.
- Cluster de servicios comunes para capacidades transversales (observabilidad, secretos, herramientas de plataforma, etc etc..).
- Clusteres de gestion y gobierno para operacion multicluster y politicas globales.
- Separacion explicita entre servicios criticos y no criticos para acotar impacto operativo.

## Escala objetivo de referencia

- Evolucion desde cluster monolitico actual hacia una flota de clusters menores.
- Referencia de trabajo discutida: alrededor de 30 clusters totales con 7-8 productivos segun avance del programa.
- Dimensionamiento base repetible por cluster para simplificar operaciones, upgrades y recuperacion.

## Criterios de diseno

- Escalabilidad horizontal mediante adicion de clusters y no por sobrecarga de uno solo.
- Portabilidad y minimizacion de lock-in con estandares abiertos.
- Compatibilidad con operacion hibrida y evolucion progresiva a nube.
- Gobernanza auditable en un entorno regulado.
