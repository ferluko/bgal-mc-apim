# 9.1 Operating model de plataforma (roles, ownership, capacidades)

El operating model objetivo se define como plataforma-producto: un modelo con ownership explicito, capacidades estandarizadas y gobierno transversal para ejecutar MC y APIM como un solo programa tecnico. La prioridad es pasar de coordinacion por excepcion a operacion predecible por dominios.

## Modelo de roles propuesto

- **Platform Engineering:** owner de day 0/day 1, componentes base y contratos de plataforma.
- **Seguridad:** owner de politicas transversales de IAM, secretos, cifrado y cumplimiento.
- **Redes/Comunicaciones:** owner de conectividad, balanceo, DNS y patrones de enrutamiento.
- **SRE/DevOps:** owner de day 2, confiabilidad, incident response y evolucion de pipelines.
- **Equipos de producto:** owner de servicios, contratos API y SLO funcionales.

## Capacidades centralizadas

- Provision de clusters y baseline de seguridad/observabilidad.
- Gobierno de politicas globales y trazabilidad de cambios.
- Libreria de patrones tecnicos reutilizables (runtime, red, seguridad, entrega).
- Operacion de componentes compartidos y continuidad inter-cluster.

## Capacidades delegadas con guardrails

- Configuracion funcional de servicios y rutas de aplicacion.
- Parametros operativos por dominio dentro de umbrales definidos.
- Evolucion de APIs de negocio sobre estandares comunes de plataforma.
- Gestion del ciclo de release de equipos sobre pipelines homologados.

## Cambios organizacionales a ejecutar

- Formalizar matriz RACI por dominio tecnico y por nivel de criticidad.
- Crear gobierno tecnico transversal con cadencia ejecutiva y tecnica.
- Establecer metricas comunes de flujo, confiabilidad y cumplimiento.
- Reducir dependencias de personas clave mediante runbooks y automatizacion.

## Beneficios esperados

- Menor friccion entre areas y mejor tiempo de entrega de capacidades.
- Mayor previsibilidad operativa para cambios e incidentes complejos.
- Mejor alineacion entre estrategia de negocio y ejecucion tecnica.
- Escalabilidad del modelo multicluster sin multiplicar complejidad organizacional.
