# 11.3 Monitoreo de experiencia de aplicacion y dependencias

La estrategia de monitoreo de aplicaciones se orienta a conectar salud tecnica con impacto funcional en usuario final. El objetivo es construir visibilidad end-to-end de dependencias para priorizar mejoras por valor de negocio, no solo por metricas de infraestructura.

## Dependencias aplicacion-aplicacion

- Estandarizar mapas de dependencias de microservicios por dominio.
- Correlacionar errores, latencia y saturacion entre servicios encadenados.
- Detectar degradaciones de flujo antes de que se propaguen.
- Integrar dependencias en planes de capacidad y resiliencia.

## Dependencias API y gateways

- Medir experiencia de consumo por endpoint, cliente y politica aplicada.
- Conectar trafico externo con comportamiento interno de servicios.
- Identificar puntos de corte en trazabilidad y resolverlos por arquitectura.
- Publicar indicadores de calidad API con impacto operativo y comercial.

## Dependencias con legacy/core

- Instrumentar interacciones OpenShift-legacy para visibilidad de punta a punta.
- Medir latencia y error en cruces de dominio tecnologico.
- Separar causa de degradacion entre red, plataforma y backend.
- Usar telemetria para priorizar secuencias de modernizacion.

## Dependencias con usuario final

- Incorporar monitoreo sintetico y real de experiencia en canales criticos.
- Relacionar eventos tecnicos con degradacion percibida por clientes.
- Definir umbrales de experiencia para alertado proactivo.
- Integrar indicadores de journey en tableros de confiabilidad.

## Beneficios esperados

- Mejor priorizacion de incidentes por impacto real en negocio.
- Menor tiempo para identificar causa de degradacion funcional.
- Decisiones de arquitectura basadas en dependencia comprobada.
- Incremento sostenido de calidad de servicio para usuarios y canales.
